\chapter{引言}

\section{研究背景}
哈希表是一种重要的数据结构，在数据库系统、操作系统、编译器以及各种键值存储系统中具有广泛应用。通过哈希函数将关键字映射到数组位置，从而实现高效的数据查询、插入和删除操作。长期以来，哈希表研究的核心问题之一是时间复杂度与空间复杂度之间的权衡问题。传统哈希表在保证常数时间查询的情况下，
往往需要通过降低负载因子（即增加未使用存储空间）来减少冲突，从而保证查询效率。论文 《On the Optimal Time/Space Tradeoff for Hash Tables》 提到，在常数时间操作的前提下，每个元素通常需要额外的空间开销。 同时提出了一种新的哈希表结构，证明可以在保持查询 O(1) 时间复杂度的同时，将插入和删除操作的时间复杂度从O(1)增加到O(k)，但是每个元素的额外空间开销可以从 $O(loglog n)$ 比特降低到 $O(log^{ (k) }n)$ 比特，在时间与空间之间实现更优的权衡。所以对该理论结构进行实现并在实际系统中进行实验验证，不仅能够加深对哈希表理论的理解，也能够探索其在数据库系统等实际场景中的应用价值。

\section{目标与范围}
围绕论文《On the Optimal Time/Space Tradeoff for Hash Tables》中的理论目标，系统阐述从“探针复杂度—交换成本”模型到完整哈希表的三层架构：微观层（Cubby 内的 k-kick 树与元数据）、路由层（设施内的 MiniArray \(D\) 与 Local Query Router）、宏观层（多 tier 小隔间、tail 不变量、后台重建与扩容缩容）。在此基础上，说明本课题\textbf{当前仓库}的工程形态：以 C++20 单进程 HTTP 服务（\texttt{cpp-httplib}）暴露实验接口；以 MySQL 持久化键集与 \texttt{otsh\_op\_metrics} 等指标表，并在服务端自动建表；另提供无 MySQL 依赖的 \texttt{otsh\_perf} 与 \texttt{otsh\_tests} 便于本地验收。

\section{主要贡献}
\begin{itemize}
  \item 将球与槽模型、k-kick 树分层结构、级联驱逐与 \(O(k)\) 交换上界整理为第 \ref{ch:bg} 章，并与 arXiv:2111.00602 的折中曲线表述对齐；
  \item 将 MiniArray、Local Query Router 独立为第 \ref{ch:design} 章的前置节，再给出 Cubby、Facility、全局状态与双表迁移/重建调度的职责划分及插入/查询/删除主路径；
  \item 阐述分布不变量 \(t_j\)、创建/销毁重建及其均摊意义，以及商压缩下键的重建思路；
  \item 归纳面向实验验证的 MySQL 关系模式（快照、cubby、tier 统计、\texttt{otsh\_op\_metrics}）与内存缓冲批量落库策略。
\end{itemize}

\section{论文结构}
全文依“底层驱逐上界—中间层紧凑编码—顶层组装”展开：第 \ref{ch:bg} 章从球与槽模型出发，形式化探针与交换成本，并归纳 k-kick 树分层驱逐与文献给出的折中上界；第 \ref{ch:design} 章先给出 MiniArray 与 Local Query Router 两类紧凑构件，再给出 Cubby、Facility 与全局扩容/重建流程及商压缩键重建；第 \ref{ch:impl} 章对应 C++/HTTP 实现与代码映射；第 \ref{ch:exp} 章给出 MySQL 快照与 \texttt{otsh\_op\_metrics} 观测设计及典型分析查询；第 \ref{ch:concl} 章总结并讨论局限与展望。附录给出关系模式与运行配置等补充材料。
