\chapter{引言}

\section{研究背景与问题动机}
哈希表因其高效的键到槽位映射能力，被广泛用于数据库索引、缓存系统、键值存储与网络服务等场景。工程实践中，哈希表通常需要在高负载因子下运行以提升空间利用率，但这会导致冲突概率上升，从而增加查询探测次数、带来不可预测的尾延迟，并放大底层存储或内存层面的访问开销。

在“以探测次数衡量查询代价”的模型下，经典开放定址哈希表在高负载时往往需要多次探测才能完成一次查询；而一些更强的结构（例如通过置换或多选择策略）可以在一定空间开销下显著降低查询探测次数。如何在可控空间代价下，保证查询只需常数次探测，是哈希表理论与工程实现共同关注的问题。

\section{研究目标与本文工作}
本文面向论文《On the Optimal Time/Space Tradeoff for Hash Tables》提出的结构思想\cite{pagh2021otsh}，复现其关键性质并完成工程化实现。该结构的核心在于：将槽位按全局桶（bin）划分，并在桶内进一步划分为若干 mini-bin 与常数大小的回退区（fallback）。对任意键的查询只需访问两个连续下标区间：其偏好 mini-bin 对应的区间，以及同一 bin 的 fallback 区间，从而将一次查询在工程意义上约束为 \(1\sim 2\) 次区间级探测。

为处理插入冲突，本文实现了有界深度的置换插入（k-kick）：当偏好 mini-bin 满时，按确定性规则选择踢出槽位并递归重排，递归深度超过阈值 \(k\) 时将元素保留在 fallback 兜底路径，保证插入过程可终止。

\section{系统实现概览与贡献}
本文实现的系统以 MySQL 为持久化存储与并发控制载体\cite{mysqlmanual}，将逻辑上的一维槽位数组映射为以全局下标为主键的关系表行；后端采用 C++20 构建 HTTP 服务并提供实验接口（基于 \texttt{cpp-httplib}\cite{cpphttplib} 与 \texttt{nlohmann/json}\cite{nlohmannjson}），前端采用 Vue 3\cite{vue3} 提供交互式操作与可视化展示。本文的主要贡献包括：
\begin{itemize}
  \item 给出 OTSH 结构在数据库场景下的槽位表示、参数派生与查询/插入/删除流程；
  \item 实现以“区间扫描次数”为度量的探针统计，支持查询与插入路径的可观测性（trace 与统计接口）；
  \item 设计并实现在线扩容流程，通过双表迁移与原子换表维持系统长期可用性；
  \item 提供可复现实验接口与前端实验面板，支持探针次数、fallback 占比与踢出深度分布等分析。
\end{itemize}
