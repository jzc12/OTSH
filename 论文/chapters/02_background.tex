\section{球与槽的在线分配}
\label{sec:balls-bins}
设共有 \(n\) 个槽，任意时刻占用球数不超过 \(n\)。每个球 \(x\) 在插入前即固定一条随机探针序列，球只能被安置在该序列中的某一槽；允许在插入或删除时移动其他球。评价指标包括：
\begin{itemize}
  \item 当前所有球的探针复杂度的平均值（与额外空间、查询路径相关）：若球 \(x\) 最终落在其探针序列的第 \(i\) 项，则探针复杂度可理解为 \(\Theta(1+\log i)\) 量级，对应为记录“使用了序列中第几个位置”所需的约 \(\log i\) 比特；
  \item 单次插入或删除过程中被移动的球的个数之和（交换成本，与操作时间相关）。
\end{itemize}
最优折中构造即是在两类指标间给出可证明的上界与（在一定结构类上的）匹配意义下的紧界。

\section{开放寻址与探针复杂度}
在开放寻址哈希表中，键 \(x\) 的查找沿探针序列 \(h_1(x),h_2(x),\ldots\) 依次检查槽位，直至命中或判定不存在。上一节的“球与槽”模型即其动态版本：插入与删除可重排已有键，使“落槽位置”不再限于首次空位。当表在高负载因子下运行时，冲突加剧，探针序列变长；若同时要求插入/删除代价可控，则往往需要在“移动已有元素”（交换）与“压低探针指标”之间权衡。

\section{k-kick 树：分层 bin 与优先级驱逐}
在 Bender 等人的 OTSH 构造中\cite{pagh2021otsh}，\(n\) 个槽被组织为深度 \(k+1\) 的分层 bin 树：第 \(0\) 层为全长数组，更深层 bin 尺寸按迭代对数迅速减小（本实现取各层尺寸为 \(\Theta\bigl((\log^{(i)} n)^6\bigr)\) 量级，与文献分层尺度同阶）。每个球在插入前随机确定：其各层祖先 bin \(g_0(x),\ldots,g_k(x)\)；最高允许深度 \(s(x)\in[0,k]\)，且 \(\Pr[s(x)<i]=1/(\log^{(i)} n)^2\)，用于定义优先级；以及沿 bin 从深到浅拼接得到的探针遍历顺序。

插入时自最深允许层向下扫描，寻找第一个“未饱和”的祖先 bin：饱和指 bin 已满且其中所有球的优先级均不低于该层。若 bin 未满则直接放入；若满但未饱和，则其中必存在来自更浅层的低优先级球，随机踢出一枚并由被踢球在其所属层继续同样规则，形成级联驱逐。每层至多踢出一次，故单次插入交换次数 \(\le k+1\)。在随机模型下，平均探针复杂度可达到 \(O(\log^{(k)} n)\) 量级；每将插入/删除时间增加一个加法常数，浪费比特可指数下降，这是原文的核心折中曲线\cite{pagh2021otsh}。本文不重新证明该曲线，而以此为上界目标组织工程结构。

\section{本章小结}
本章从在线“球与槽”出发，固定了探针复杂度与交换成本两个量，并给出 k-kick 树的分层、优先级与级联驱逐机制，以及 \(O(k)\) 次交换的上界直觉。第 \ref{ch:design} 章将在同一章内先给出 MiniArray 与 Local Query Router 两类紧凑构件，再给出 Cubby、Facility 与全局插入/查询/删除流程。
