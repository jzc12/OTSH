\chapter{问题模型与理论基础}
\label{ch:theo_base}

\section{探针复杂度与球槽分配模型}
开放寻址哈希表的本质可抽象为经典的球与槽在线分配问题。给定固定数量数组槽位，每个槽仅能容纳一个元素，支持元素动态插入与删除，且元素总数不超过槽位上限。每个键经由哈希函数生成一组固定随机探针序列，元素仅允许放置在自身探针序列的任意候选槽位中。

若元素最终落于探针序列第 \(i\) 个位置，则定义其探针复杂度为 \(\Theta(1+\log i)\)，物理意义为直接定位该元素所需记录位置索引的比特开销，表征哈希表元数据的平均空间代价。在此模型下衍生两个核心优化指标：平均探针复杂度对应哈希表整体额外空间开销，交换成本对应插入删除过程中迁移元素的数量，直接决定操作时间复杂度。哈希表优化的核心目标，即是在探针空间开销与元素迁移时间开销之间实现时空最优折中。

\section{k-kick 树分层驱逐思想}
k-kick 树并非物理意义上的树形存储结构，而是一套面向数组槽位的分层划分与优先级驱逐逻辑框架。其核心思想是将全局槽位划分为 \(k+1\) 个逻辑层级，各层级隔间 Cubby 尺寸随层级加深呈指数级递减，形成由大到小的嵌套分层结构，深层隔间对应元素更偏好的局部存储位置，具备更高存储优先级。

元素在插入前通过随机哈希预先确定所属层级隔间谱系、最大允许存储深度优先级，以及由深层至浅层遍历的专属探针序列。插入过程采用贪心优先级策略，从最深层级向上逐层检索未饱和隔间；若隔间已满但未饱和，则驱逐内部低优先级元素，将当前元素置入空闲槽位，被驱逐元素向上层递归寻找合法位置，形成可控级联驱逐。

由于层级总数被参数 \(k\) 限定，单次插入的元素迁移次数被严格约束，以 \(O(k)\) 交换成本换取 \(O(\log^{(k)}n)\) 的平均探针复杂度，通过有限元素迁移显著压低整体元数据空间开销。

\section{紧凑元数据存储核心思想}
为在常数时间、近最优空间下管理分层哈希表的路由信息与控制元数据，系统引入两类基础抽象组件：MiniArray 与 Local Query Router。

MiniArray 面向各 Cubby 内部变长元数据条目设计，以内存 B 树结合查表加速思想，实现变长比特条目的紧凑存储与均摊常数时间读写，整体空间开销与所有条目总比特数线性相关，避免固定长度预分配带来的空间冗余。

Local Query Router 面向小规模键集合，基于二进制前缀树压缩编码思想，将键哈希为随机比特串后仅保留区分唯一前缀，整棵前缀树连同槽位元数据被编码为连续比特流。通过批量比特查表跳转，实现键到 Cubby 与内部槽位的常数时间路由映射，且整体编码空间维持对数比特量级。两类组件共同支撑了哈希表在微观元数据存储与宏观键路由上的低空间、常数时间访问能力。

\section{分层架构整体设计思想}
本文所依托的最优哈希表采用 \textbf{Facility–Cubby–Slot} 三级分层架构，从宏观组织、局部存储到底层槽位形成完整逻辑体系。

以 Cubby 作为基本管理单元，每个 Cubby 等价为独立的 k-kick 树逻辑实例，内部集成实际数据存储、变长元数据编码以及空闲槽快速检索结构，承载元素放置、驱逐与局部元数据维护能力。上层以 Facility 聚合多个 Cubby，通过顶层路由 MiniArray 与本地路由器完成全局键到设施、再到目标 Cubby 的两级寻址；同时引入多层级 Cubby 与 Tail 尾隔间机制，仅保留尾隔间承载插入删除波动，其余隔间维持满容量不变量，保证系统负载稳定性。

为适配元素规模动态变化，架构引入分布不变量约束各层级 Cubby 的数量配比，通过后台拆分与合并操作平摊重建开销，使扩容、缩容与结构调整均能维持均摊常数时间复杂度。插入、查询、删除等核心操作均遵循统一路由寻址、局部隔间处理、元数据更新与不变量维护的逻辑流程，将底层 k-kick 树规则与上层路由架构有机结合。

\section{商压缩的信息论最优思想}
传统哈希表完整存储原始键的全部比特位，存在固有空间冗余。商压缩的核心思想是利用哈希随机置换的寻址特性，将键的高位比特融合进路由寻址逻辑中隐式存储，无需在数据区重复保存。

通过将键哈希置换后的低位作为设施与路由索引，仅在底层 Cubby 数据区存储键的剩余高位与负载数据；同时在元数据中记录偏好槽位差值与分段中间比特，结合设施编号可逆向完整恢复原始哈希值与真实键。该方式省去路由寻址所依赖的高位比特存储，使单键存储开销逼近信息论下界，叠加元数据与路由的对数级均摊开销，整体哈希表空间复杂度达到理论最优水平。

\section{算法层次全景总结}
整套最优哈希表可归纳为三层逻辑协同架构：
\begin{itemize}
  \item 微观 Cubby 层：依托 k-kick 树实现元素分层放置与优先级驱逐，配合空闲槽检索与 MiniArray 元数据编码，管控单隔间内的空间开销与元素迁移规则；
  \item 路由映射层：基于 Local Query Router 与顶层路由数组，实现键到目标隔间与槽位的常数时间寻址，匹配底层探针复杂度的空间约束；
  \item 宏观 Facility 层：通过多层级 Cubby、Tail 机制与后台重建不变量维护，支撑动态扩容缩容与负载均衡，保证所有操作时间受控于层级参数 \(k\)，整体空间复杂度逼近信息论最优下界。
\end{itemize}
