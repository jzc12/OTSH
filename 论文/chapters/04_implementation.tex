\chapter{实现要点}
\label{ch:impl}

\section{技术栈与交付物}
本仓库以 CMake（要求 $\geq$3.20）组织构建：核心为 C++20 可执行文件 \texttt{otsh\_backend}（HTTP 服务）、\texttt{otsh\_tests}（内存表回归并可写 MySQL 测试日志）、\texttt{otsh\_perf}（纯内存插入/查询/删除延迟与进程内 \texttt{Metrics} 汇总，不链接 MySQL）。HTTP 层依赖单头库 \texttt{cpp-httplib}\cite{cpphttplib} 与 \texttt{nlohmann/json}\cite{nlohmannjson}；可选置于 \texttt{third\_party} 或通过 FetchContent 获取。持久化与观测走 MySQL C API（\texttt{libmysqlclient}/MariaDB 兼容），由 \texttt{src/storage\_mysql.cpp} 在首次连接时 \texttt{CREATE TABLE IF NOT EXISTS} 分析侧各表。

\section{运行形态与并发模型}
\texttt{otsh\_backend} 内持有一个全局 \texttt{std::mutex}（\texttt{service\_mu}），对 \texttt{/api/init} 与所有写路径（插入/删除及依赖表状态的实验任务）加锁，保证与当前“单 writer、可重复实验”目标一致；查询与统计在实现中同样经该锁与表状态交互，行为以代码为准。探活为 \texttt{GET /health}，返回纯文本 \texttt{ok}。进程日志默认写 \texttt{backend.log}，可通过环境变量 \texttt{LOG\_FILE} 覆盖。监听地址与端口由 \texttt{BIND\_HOST}（默认 \texttt{0.0.0.0}）、\texttt{PORT}（默认 \texttt{8080}）指定。

\section{配置与存储入口}
\texttt{TableParams}（\texttt{include/config.h}）描述初始化规模 \texttt{n}、占位参数 \texttt{k}、\texttt{load\_factor}、三枚种子 \texttt{seed1..seed3} 以及 MySQL 主机/端口/用户/密码/库名/主键表名（默认表名 \texttt{otsh\_keys}，单列 \texttt{k BIGINT UNSIGNED PRIMARY KEY}）。当前 \texttt{/api/init} 请求体主要解析 \texttt{n}、\texttt{k}、\texttt{reset}、\texttt{snapshot\_tag}，其余字段沿用上述默认值；自动化测试程序另支持 \texttt{OTSH\_MYSQL\_*} 环境变量覆盖连接信息。初始化成功后创建分析用 \texttt{snapshot\_id} 并作为后续 \texttt{otsh\_op\_metrics} 行的外键上下文。

\section{核心数据结构映射}
对外类型为 \texttt{HashTable}（\texttt{include/ht.h}，实现于 \texttt{src/ht.cpp} 的 \texttt{HashTable::Impl}）。内部以嵌套结构 \texttt{TableState} 保存当前表的 \(N\)（不小于 2 的 2 的幂）、\(K\)、\texttt{std::vector<Facility>}；另以 \texttt{std::optional<TableState> old\_} 与 \texttt{migrate\_progress\_} 实现\textbf{双表扩容/缩容迁移}：当负载超过 \texttt{load\_factor} 或键数低于 \(N/4\) 时分配新表并逐设施预算化搬迁，迁完后释放旧表。文献中的 \texttt{ResizeManager} 在仓库 \texttt{include/otsh/resize.h} 中仍为占位类型，实际迁移逻辑内联于 \texttt{HashTable::Impl}。

\texttt{RebuildScheduler}（\texttt{include/otsh/rebuild.h}）维护 \texttt{std::deque} 任务队列，在每次插入后的预算步数内执行 tier 合并/拆分等后台工作，对应设计章分布不变量触发逻辑的工程化版本。

\texttt{PermutationHash} 将 64 位键高/低各 32 位视作 Feistel 两半，使用三轮轮函数（轮密钥取自 \texttt{k1..k3}，由 \texttt{splitmix64} 与种子派生）；\texttt{k4} 与额外种子字段一并派生但\textbf{不参与}当前三轮 Feistel 轮函数，保留作扩展。逆置换 \texttt{inverse} 对称实现，用于校验与调试路径。

每个 \texttt{Facility}（\texttt{include/otsh/facility.h}）包含：按 tier 分组的二维 \texttt{std::vector}，内层元素为 \texttt{std::unique\_ptr<Cubby>}；\texttt{tail} 指针与独占 \texttt{tail\_owned}；长度为 \(K\) 的 \texttt{std::vector<Router>} 即设施内路由器数组 \texttt{D}。每个 \texttt{Cubby}（\texttt{include/otsh/cubby.h}）含 \texttt{tier}、\texttt{capacity}、\texttt{size}、商压缩载荷槽向量 \texttt{slots}（元素为 \texttt{std::optional<uint64\_t>}）、占用索引表、\texttt{FreeSlotTree free\_slots} 与 \texttt{MiniArray meta}。

\section{MiniArray 与路由器（工程形态）}
\texttt{MiniArray}（\texttt{include/otsh/mini\_array.h}）当前实现为：每条目变长比特写入连续 \texttt{buf\_}，以 \texttt{offsets\_} 与 \texttt{bitlens\_} 定位；更新允许对单 cubby 规模 \(m\) 做 \(O(m)\) 重打包，满足“polylog 规模下可接受”的原型目标，与设计章所述 packed B-树理想模型相比更偏工程折中。

\texttt{Router}（\texttt{include/otsh/router.h}，对应设计章 Local Query Router）用 64 位哈希串上的二叉 trie 加碰撞桶管理 \texttt{key\(\to\)(Cubby*, slot)}；维护编码比特流规模 \texttt{bits\_total} 供空间统计。插入与删除后允许整表重建 trie/编码（元素数为 polylog 时开销可控），与“均摊常数跳转”的理论目标之间留有优化空间。

\section{操作主路径与指标}
插入/查询/删除主流程与第 \ref{ch:design} 章叙述一致：经 \(\pi(\texttt{key})\) 与设施、桶索引定位 \texttt{D[b]}，在 tail cubby 上执行 k-kick 相关逻辑并维护路由器与商压缩元数据。每次操作可将 \texttt{probe\_count}/\texttt{kick\_count}/\texttt{latency\_ns}/\texttt{cubby\_tier} 等封装为 \texttt{SqlMetricRow}，经 \texttt{IStorage::analytics\_enqueue\_metric} 进入缓冲；缓冲长度达到 1000 或显式 \texttt{analytics\_flush\_metrics(true)} 时，以最多 500 行一批多值 \texttt{INSERT} 写入 \texttt{otsh\_op\_metrics}。进程内另有一套原子计数器 \texttt{Metrics}（\texttt{include/metrics.h}）汇总吞吐、移动步数、延迟直方图与重建/扩容事件，\texttt{otsh\_perf} 直接读取其快照。

\section{HTTP 与分析接口（摘要）}
除 \texttt{/api/init} 外，实现还提供 \texttt{/api/insert}、\texttt{/api/query}、\texttt{/api/delete}、\texttt{/api/stats}；分析侧含 \texttt{/api/analytics/snapshots}、\texttt{/api/analytics/summary}、\texttt{/api/analytics/structure\_dump}；长任务轮询 \texttt{/api/jobs/get}；参数扫描实验 \texttt{/api/experiment/o1\_vs\_n}、\texttt{/api/experiment/ok\_vs\_k}。各路径的请求/响应 JSON 字段与错误码以仓库源码为准，附录仅补充库表与环境变量。
