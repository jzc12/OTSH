\chapter{总结与展望}
\label{ch:concl}

\section{工作总结}
本文对 OTSH 型最优时间/空间折中哈希表进行了自洽的论文化整理：第 \ref{ch:bg} 章从球与槽模型与探针复杂度出发，说明 k-kick 树如何通过分层 bin、随机优先级与级联驱逐在 \(O(k)\) 次交换内实现文献意义下的折中目标；第 \ref{ch:design} 章将 MiniArray 与 Local Query Router 作为独立中间层叙述后，再给出 Cubby、Facility、全局置换与商压缩下的插入/查询/删除主路径及分布不变量。实现层面概述了 C++/HTTP、MySQL 快照与指标、Vue 前端及批量写入策略，使结构行为可被记录与可视化。

\section{局限与后续工作}
\begin{itemize}
  \item 理论命题的严格概率陈述与证明仍以原文为准\cite{pagh2021otsh}，本文未给出形式化引理证明链；
  \item 工程常数（\(K\)、tier 边界、缓冲批次、迁移调度）对尾延迟与空间占用的影响需系统化基准测试；
  \item MiniArray 与 Trie 路由器的查表宽度、缓存友好性与并行安全可进一步专项优化；
  \item 可探索将快照与指标导出与离线仿真器对接，以分离算法与 I/O 因素。
\end{itemize}

\section{结束语}
将分层驱逐、紧凑编码与多 tier 重建统一在同一实现中，有助于在数据库与服务化场景下验证“理论最优结构”的实际轮廓。希望本文为后续开发与论文撰写提供一致术语与结构索引。
