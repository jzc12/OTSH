\begin{abstract}
  哈希表是高性能存储系统中最基础的数据结构之一。为在高负载下同时获得接近常数的查询代价与较高的空间利用率，本文面向论文《On the Optimal Time/Space Tradeoff for Hash Tables》提出的结构，完成其在工程环境中的复现与系统化实现。该结构以全局分桶（bin）与桶内分块（mini-bin）组织槽位，并为每个桶设置常数大小的回退区（fallback）；对每个键仅需检查“偏好 mini-bin 区间”和“fallback 区间”两段连续下标范围即可完成查询，从而在工程意义上将一次查询约束为 1–2 次区间级探测。为处理冲突，本文实现了有界深度的置换插入（k-kick），在深度超过上界时将元素留在回退区以保证插入过程可终止。

  系统实现采用 MySQL 作为存储与并发控制载体，将槽位映射为以全局下标为主键的关系表行，并存储 64 位键值与 16 位指纹以加速过滤；后端使用 C++20 构建 HTTP 服务，前端基于 Vue 3 提供交互式实验界面，支持初始化、插入、查询、删除、在线扩容以及探针次数、回退占比与踢出深度分布等统计接口。实验与工程实践表明，该实现能够在较高负载因子下保持稳定的查询探测次数，并通过在线扩容机制维持长期运行的可用性。本文工作为理论最优时间/空间折中哈希表在数据库与服务化场景中的工程落地提供了可复现的参考实现与实验支撑。
\end{abstract}

\begin{abstract*}
  Hash tables are a fundamental building block in high-performance storage systems. To achieve near-constant lookup cost under high load while maintaining good space efficiency, this thesis reproduces and engineers the core design proposed in *On the Optimal Time/Space Tradeoff for Hash Tables*. The data layout partitions slots into global bins and intra-bin mini-bins, and augments each bin with a constant-size fallback area. For any key, a lookup only scans two contiguous index ranges—the preferred mini-bin segment and the bin’s fallback segment—so a query is bounded to about one to two range-level probes in practice. To resolve collisions during insertion, we implement bounded-depth displacement (k-kick); when the recursion depth exceeds the preset limit, the key is kept in the fallback path to guarantee termination.

  Our system uses MySQL as the persistent storage and concurrency control substrate, representing the logical slot array as table rows keyed by a global index. Each occupied slot stores the full 64-bit key/value and a 16-bit fingerprint for efficient filtering. The backend is a C++20 HTTP service, and the frontend is a Vue 3 application that exposes interactive operations and experiments, including initialization, insert/find/erase, online resizing, and statistics such as probe counts, fallback occupancy, and kick-depth histograms. Empirical results and engineering observations show that the implementation maintains stable lookup probes at high load factors and remains usable over long runs via online resizing. This work provides a reproducible reference implementation and experimental basis for deploying the theoretical time/space optimal hash table in database-backed and service-oriented settings.
\end{abstract*}
