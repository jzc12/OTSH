\begin{abstract}
  开放寻址哈希表可归结为在线“球与槽”分配：每个键沿固定随机探针序列寻找槽位，插入与删除可移动已有键；平均探针复杂度与交换成本构成一对核心折中。本文依“底层驱逐上界—中间层紧凑编码—顶层组装”的顺序整理 OTSH 型构造在工程中的展开方式：以 k-kick 树在分层 bin 上做贪心优先级驱逐，使单次插入交换次数不超过 \(O(k)\)，并在随机模型下将平均探针控制在 \(O(\log^{(k)} n)\) 量级；继而给出 MiniArray 与局域查询路由器，再说明以 Cubby 封装 polylog 规模的存储数组、变长元数据与空闲槽位树，以 Facility 聚合约 \(K=\mathrm{polylog}\,N\) 个键的路由 MiniArray \(D\) 与多 tier 小隔间，并用唯一 tail 维持“除尾外全满”的饱和度不变量；以分布目标 \(t_j\) 触发创建/销毁重建与双表扩容缩容；以商压缩隐式携带路由高位，使载荷空间接近信息论下界。

  实现上采用 C++20 单进程 HTTP 服务（\texttt{cpp-httplib}），MySQL 持久化键集与 \texttt{otsh\_op\_metrics} 操作指标，并附带无数据库依赖的 \texttt{otsh\_perf} 基准与 \texttt{otsh\_tests} 回归；热路径对指标采用内存缓冲、达千条或显式 flush 时批量写入，结构快照由 \texttt{/api/analytics/structure\_dump} 等接口触发。本文为当前仓库中与理论构件对齐的模块划分、表结构与运行配置提供可直接对照的叙述。
\end{abstract}

\begin{abstract*}
  Open-addressing hash tables can be viewed as an online balls-into-bins process: each key follows a fixed random probe sequence, and inserts/deletes may relocate existing keys, trading average probe complexity against swap cost. Following the same bottom-up organization (eviction bounds, compact metadata, global assembly), this thesis presents an engineering-oriented account of an OTSH-style construction: a k-kick tree performs greedy prioritized eviction over layered bins, bounding swaps per insert by \(O(k)\) while targeting \(O(\log^{(k)} n)\) average probes in the random model; a Cubby packs a polylog-sized slot array, variable-length MiniArray metadata, and a free-slot tree; a Facility aggregates roughly \(K=\mathrm{polylog}\,N\) keys via a routing MiniArray \(D\) and multi-tier cubbies with a unique tail enforcing an almost-everywhere-full saturation invariant; tier-count targets \(t_j\) trigger split/merge rebuilds alongside doubled/halved resizing; quotienting hides routing bits to approach information-theoretic space.

  The implementation ships a C++20 single-process HTTP server (\texttt{cpp-httplib}), MySQL for the key table and an \texttt{otsh\_op\_metrics} fact table, plus a MySQL-free \texttt{otsh\_perf} benchmark and \texttt{otsh\_tests}; hot paths buffer per-operation metrics and bulk-insert on threshold or explicit flush, while structural dumps are triggered via \texttt{/api/analytics/structure\_dump}. The thesis records terminology and schema aligned with this repository for reproducible experimentation.
\end{abstract*}
