\begin{abstract}
哈希表是计算机科学中应用最广泛的基础数据结构之一，如何在查询时间与存储空间之间取得最优权衡，是哈希表领域长期存在的核心问题。
本文以《On the Optimal Time/Space Tradeoff for Hash Tables》为理论基础，针对论文的关键算法与数据结构进行系统实现，完成
从探针复杂度优化、元数据紧凑编码到完整哈希表系统的工程化落地。
首先，针对探针复杂度问题，本文实现了基于 k-kick tree 的 balls-to-slots 放置策略。通过分层桶结构设计与贪心优先级驱逐机制，
将单次插入与删除操作的元素移动次数控制在 $O(k)$ 复杂度，同时在随机哈希模型下将平均探针复杂度降至 $O(log^{(k)}n)$，实现了理
论最优的时间–空间折衷。
其次，针对哈希表查询路由元数据占用空间过大、访问效率低的问题，本文依据论文思想设计并实现 MiniArray 与 Local Query Router 
两种紧凑数据结构。依托内存 B - 树与查找表加速技术，实现元数据的紧凑存储与常数时间操作，将单键元数据开销降至 $O(log log n)$ 比
特，显著提升空间利用率。最后，基于上述核心组件构建完整的增强开放寻址哈希表，设计 Cubby 存储单元、Facility 管理结构以及动态扩容
与重建机制，结合商压缩技术实现键的高效存储与还原，并通过分布不变量 $t_j $控制分层桶的创建与合并，最终实现支持 $O(1) $查询、$O(k)$ 
插入删除、动态可伸缩且空间浪费逼近信息论下界的高效哈希表。
本文工作将前沿理论成果转化为可运行、可验证的工程实现，验证了最优时间–空间权衡哈希表的可行性。
\end{abstract}

\begin{abstract*}
  Hash tables are a core data structure in computer science, with the optimal trade-off between time and space remaining a central open problem for over sixty years. This thesis, based on the theoretical results from arXiv:2111.00602, focuses on the core content of chapters 3, 4, and 5, completing the design of the optimal solution for the probe complexity problem, the query routing metadata encoding, and the implementation of the optimal enhanced open-addressing hash table.
  First, for the probe complexity problem, the k-kick tree balls-to-slots scheme is designed, through greedy prioritized eviction and layered bin structure, bounding the single insertion swap count to $O(k)$ complexity, and controlling the average probe complexity to $O(log^{(k)}n)$ level in the random model.
  Second, for the query routing metadata encoding problem, the MiniArray and Local Query Router two compact components are designed, through memory B-tree and lookup table acceleration, compressing the bit length of each element to $O(log log n)$ level, while supporting O(1) read and write.
  Finally, for the implementation of the optimal enhanced open-addressing hash table, the Cubby, Facility, and global expansion/reconstruction process and quotient compression key reconstruction are designed, through the distribution invariant $t_j$ triggering create/destroy reconstruction, completing the implementation of the optimal enhanced open-addressing hash table.
  This work has converted frontier theoretical results into runnable and verifiable engineering implementations, validated the feasibility of the optimal time–space trade-off hash table, and provided a complete implementation scheme and practical reference for the design of high-performance, low-space-overhead dictionary structures.
\end{abstract*}
