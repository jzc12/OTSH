\begin{abstract}
开放寻址哈希表在工程里很常见，但在高负载下既要省内存、又要控制插入删除的扰动，并不容易。Bender 等人的论文《On the Optimal Time/Space Tradeoff for Hash Tables》给出了一条理论路线：用有限次“踢键”换更紧凑的表示。本毕业设计以该论文为主要参照，用 C++ 写了一个可运行的原型：键先经可逆置换再分区；每个 Facility 内用 Router 做局部定位；槽里做商压缩，变长恢复信息放在 MiniArray；扩缩容采用新旧两张表逐步搬迁。

实现上对原文有多处简化：k-kick 只在单个 tail cubby 里用有界随机探针和踢出链近似；MiniArray 与 Router 在更新时允许整段重打包或重建 trie，以保证代码清晰和指标好记。功能测试覆盖插入、查询、删除和重复插入；另有独立微基准查看热路径延迟。实验表明，在默认参数下查询延迟稳定在纳秒量级，插入阶段因重打包和扩容会慢一些。本工作重在把论文里的主要模块串起来并说明与理论的差距，而不是证明渐近最优性。
\end{abstract}

\begin{abstract*}
This undergraduate thesis implements a simplified prototype of the hash table construction in ``On the Optimal Time/Space Tradeoff for Hash Tables.'' The code uses a reversible permutation for routing, per-facility routers to locate cubbies and slots, quotient-compressed payloads with variable-length metadata, and two-table incremental resizing.

Several parts of the theory are only approximated: k-kick style behavior is reduced to a bounded kick chain inside the tail cubby, and metadata structures favor straightforward updates over the tightest asymptotics. Functional tests check correctness; a small micro-benchmark reports latency and counters. The goal is a working, observable system and a clear statement of engineering tradeoffs, not a full reproduction of every formal guarantee in the paper.
\end{abstract*}
