

## 论文原理

### 最底层的核心问题：探针复杂度

任何增广开放寻址哈希表最终都可以归结为一个“球与槽”的在线分配问题。

#### 1.1 问题定义

假设有  n 个槽，每个槽最多放一个球。球会动态地插入、删除，总数不超过  n。每个球 x 有一个固定的、随机的探针序列： 
h₁(x), h₂(x), h₃(x), ...
它只能放在这个序列里的某个槽中。如果球  x 最终落在 hᵢ(x)，我们称它的探针复杂度为 Θ( 1 + log i )。

这个值直观上就是“为了在查询时直接跳到这个球的位置，最少需要额外记录的比特数”（必须记录它用的是序列中的第几个位置，而存储索引 i 需要大约 log i 比特）。

球与槽方案要解决的就是在线地给每个球指派一个合法的槽位，并且允许在插入/删除时移动已有的球。两个性能指标：

- 平均探针复杂度：当前所有球的探针复杂度的平均值，对应最终哈希表的每个键的额外空间开销。
- 交换成本：一次插入/删除过程中移动的球的总数，对应操作的时间复杂度。

目标就是在这两者之间找到最优的折中。

#### 1.2 上界：k-kick 树

> k-kick tree 是 逻辑结构，不是物理结构

k-kick 树是一套分层优先级驱逐的方案，它用$ O(k)$ 的交换成本换来了 $O(log^{(k)} n)$ 的平均探针复杂度。

##### 分层结构
把 n 个槽组织成一棵深度为 k+1 的树，从第 0 层到第 k 层，每一层把数组划分成大小呈指数级递减的cubby：

- 第 0 层：整个数组，大小 s₀ = n。
- 第 1 层：bin 大小 $s₁ = Θ((log n)^6)$ 
- 第 2 层：bin 大小 $s₂ = Θ((log log n)^6)$ 
- ……
- 第 k 层：bin 大小$ sₖ = Θ((log^{(k)} n)^6)$ 

这些 cubby之间有父子关系：每个第 i 层的 cubby包含在第 i-1 层的某个 bin 中。这就像一棵树，越往下叶子越小。

##### 球的随机偏好
每个球 x 在插入前就通过哈希函数随机确定了三件事：

1.  一个叶子 cubby：随机选取第 k 层的一个 cubby（称其深度 k 的 cubby）。由此也就确定了它在每一层的祖先 cubby我们用 $g₀(x), g₁(x), …, g_k(x)$ 表示球 x 在各层的 cubby。
2.  一个最高允许深度：随机分配一个值 s(x) ∈ [0, k]，满足 $ Pr[s(x) < i] = 1 / (log^{(i)} n)^2$ 。这个值决定了球的“优先级”：只允许被放在深度 s(x) 或更浅（即 cubby 更大）的层。深度越深（cubby 越小）优先级越高（因为越靠近叶子）。
3.  探针序列：它的探针序列按从深到浅的顺序遍历其所属的 cubby：先遍列 $g_k(x) $中的所有槽，然后是 $g_{k-1}(x)，……，$最后是 $g₀(x)$。这样，如果球最终在深度 i 的 cubby 中，它的实际探针复杂度大约为  $Θ(log s_i) = Θ(log^{(i+1)} n)$ 。

##### 插入与级联驱逐
当插入一个新球 x 时，我们不直接选择探针序列中的第一个空位，而是采用贪心 + 优先权策略：

1.  确定 x 能使用的最高深度 $min(k, s(x))$。
2.  从深度 k 开始向下扫描，找到第一个未饱和的祖先 cubby。cubby 饱和 = 满 + 所有元素深度≥当前层深度，仅未饱和才允许驱逐低深度元素
3.  假设找到的深度为 i，cubby为$ g_i(x)$。
    - 如果 cubby中有空位：直接放入。
    - 如果 cubby已满：因为 cubby未饱和，里面必定存在一个低优先级的球（深度 < i，即来自更上层 cubby 的球）。我们随机选一个这样的球 y，把它“踢”出去，把 x 放到 y 的槽里。
4.  被踢出的 y 现在需要在它的所属深度（假设为 i' < i）的 cubby  $g_{i'}(y)$ 中找一个新位置。这个过程与插入 x 类似：如果那个 cubby有空位则放入，否则再踢出一个更低优先级的球，形成级联驱逐。
5.  这个过程持续直到有一个球找到了空位（最终会因为深度 0 的 cubby是整个数组且必定有空位而终止）。

由于每次插入最多在每一层踢出一个球，而共有 k 层，所以交换成本 ≤ k+1。



### 中间层：紧凑元数据存储

现在还要高效地记录每个球在哪个槽，以及相关的控制信息。论文中用到两个基本组件：MiniArray 和 Local Query Router。

#### MiniArray：存储变长条目的紧凑动态数组

每个 cubby 里需要维护一个长度约为 polylog n 的数组，但每个元素的比特长度不同，且会动态变化。如果简单地用指针或预先分配最大空间，浪费太大。MiniArray 就是用来解决这个问题的，它的总空间与所有条目的总比特数成线性关系，同时支持 O(1) 的读写。

实现方法：内存 B-树 + 查找表加速

- 把数组看作一个 B-树，每个节点的大小恰好是一个机器字（$Θ(log n) $比特）。
- 叶子节点存储实际条目比特，内部节点存储指向孩子的偏移以及枢轴。
- 因为总条目数只有 polylog n，所以指针和枢轴都可以用 $Θ(log log n)$ 比特，依然能打包进一个字里, 树的高度是 $O(1)$。
- 所有在节点上的操作（读取枢轴、比较、提取孩子指针、分裂/合并节点）都可以通过预先构建的查找表在 O(1) 时间内完成。查找表以半个字的比特串作为输入，输出对应的处理结果，这一点利用了“四俄方法”思想。
- 当数组更新导致某个节点过长或过短时，像 B-树一样分裂或与兄弟合并，均摊 $O(1) $时间。



#### Local Query Router：为少量元素映射键到槽索引

在哈希表的顶层（Facility 的 D 数组）以及每个 cubby 的内部，我们都需要解决一个问题：给定一个键 x，立马知道它在哪个 cubby和 cubby 内的哪个槽里。因为每个 router 只管理非常少的键（最多 $O(log n / log log n) $个），可以采用一种简单的二进制 Trie + 编码方法。

结构：

- 对每个待管理的键，把它哈希成一个随机二进制串。
- 将所有键的二进制串插入一个二进制 Trie，但只存到能够与其他键区分开的最小唯一前缀。
- 在 Trie 的叶子处，存储该键对应的元数据（比如 cubby ID 和槽索引），这些值也压缩成变长比特。
- 把整棵 Trie 连同叶子值按深度优先遍历序列编码成一个紧凑的比特流。

操作：

- 查询：把查询键也哈希成二进制串，在 Trie 中沿路径走。由于 Trie 的编码是比特流，可以利用查找表一次处理多个比特（比如半个字），实现 O(1) 的跳转。
- 插入/删除：修改 Trie，更新编码比特流，同样通过查找表完成常数时间操作。

空间：由于随机二进制串的公共前缀很短，期望 Trie 的大小只与元素数量（和哈希串的随机性）有关，而不会产生巨大浪费。对于 $O(log n / log log n) $个元素，整个编码的期望长度也就在$ O(log n) $ 比特级别，且高概率不会超过$ O(log n) $比特。

这两个组件共同保证了：无论在宏观路由还是微观元数据存储上，我们都能以几乎最优的空间实现常数时间的访问。

### 顶层：组装成最优哈希表

#### 整体架构

> Facility → Cubby → Slot

#### 小隔间（Cubby）的内部设计

整个哈希表被划分为许多 cubby ，每个小隔间本质上就是一个独立的 k-kick 树实例，大小约为 polylog n。

cubby 内部包含：

- 存储数组 S：长度为 $|I|$，存放值的实际 payload（或者商压缩后的键后缀和值）。
- 元数据 MiniArray Meta：每个槽对应一个变长条目，里面编码了三类信息：
  1.  查询路由信息（ A_I）：用于 Local Query Router 找到槽位后的进一步定位。
  2.  插入元数据（ M_I）：k-kick 树所需的控制信息，例如每个槽内元素的深度、各 bin 饱和状态等。
  3.  商压缩元数据（ B_I）：差值 $g_I(x) - i$、部分高位比特，用于从槽位反推完整键。
- 空闲槽位树：一个 $O(1) $深度的位图树，能快速找到空闲槽。

#### 宏观结构：设施（Facility）与路由

为了支持动态扩容和分摊重建成本，哈希表由 N/K 个设施组成。每个设施内聚了大约 K = polylog N 个键。

- 顶层路由 D：每个设施有一个长度为 K 的 MiniArray D。对于键 x，我们先通过商压缩后的哈希值 g*(x) 的高位选择设施，低位选择 D 数组中的某个位置 b。D[b] 就是一个 Local Query Router，它存着所有落到这个位置的键的目标小隔间指针和槽索引。
- 多层小隔间与 tail：为了维持高效的重建，每个设施内的小隔间有不同的大小（tier）。大部分小隔间必须是满的，只有一个称为 tail 的 1-tier 小隔间可以不满足承受插入和删除的波动。插入一律进 tail，删除非尾小隔间时从 tail 搬一个元素过来填补，以保持“除 tail 外全满”的饱和度不变量。

#### 分布不变量与后台重建

随着元素数量的变化，各 tier 小隔间的数量必须调整，以保持整体的空间利用率和元数据大小在目标范围内。通过维护一个分布不变量： 
每个设施中 tier j 的小隔间数量应大致为 $ t_j = \frac{(log^{(j)}n)^2}{(log^{(j+1))}n^2}$。

一旦某个 tier 的数量偏离目标太远（比如少于 $t_j/2 $或多于 $3t_j$），触发后台重建：

- 创建重建（数量太少）：拿一个 tier j+1 的小隔间拆分成  $ t_j$  个 tier j 小隔间。
- 销毁重建（数量太多）：把 $ t_j $ 个 tier j 小隔间合并成一个 tier j+1 小隔间。

关键：这些重建工作被平摊到多次插入/删除操作中。例如，重建一个小隔间需要$ O(|I|)$ 时间，而触发重建的频率是每 $Θ(|I|)$ 次操作，因此每次操作只均摊$ O(1) $的额外时间。

#### 整体流程

##### 插入(Key x)
1.  应用随机置换 π：$y = π(x)$。
2.  取 $g* = y$ 的低 log N 位，拆分为设施索引  r  和路由索引 b。
3.  在设施 r 中，查询 D[b] 的路由器，确认键不存在。
4.  将键插入到设施  r  的 tail 小隔间。
    - 在 tail 内执行 k-kick 树插入，得到槽位 slot。
    - 在 S[slot] 写入键的后缀（或完整键，若未商压缩）和值。
    - 编码 MetaEntry（包含 delta、mid_bits 等）并写入 M[slot]。
    - 更新空闲槽位树。
5.  在 D[b] 的路由器中添加记录：键指纹 → (tail 指针, slot)。
6.  维护不变量：若 tail 变满，创建新的 tail。
7.  递增 n；若 n > N，触发扩容过程。

##### 查询(Key x)
1.  $y = π(x)$，提取 r 和 b。
2.  访问 D[b] 的路由器，给出小隔间 cubby 和槽位 slot。
3.  从 cubby->S[slot] 获取 payload 后缀，结合 Meta[slot] 中的 delta 和 mid_bits 以及 r 恢复出 g*，进而得到 y。
4.  应用 π⁻¹(y) 得到原键，验证一致性后返回值。

##### 删除(Key x)
1.  查询得到 cubby 和 slot。
2.  清空 cubby ->S[slot] 和 Meta[slot]，更新空闲树。
3.  从路由器中移除该键。
4.  如果 cubby  不是 tail：
    - 从 tail 中随机挑一个元素，从 tail 删除它。
    - 将该元素插入到 cubby 刚刚空出的槽位。
5.  如果 tail 变空，则销毁该 tail，并把另一个 1-tier 满小隔间提升为新 tail。
6.  递减 n；若 n < N/4，触发缩容。

#### 3.5 商压缩：如何达到信息论最优

上述流程中，我们依然存储了键的全部$ log|U|$比特。

商压缩的核心想法是：键的哈希值本身就是随机寻址的一部分，我们可以用寻址过程来隐式存储键的高位。

在商压缩模式下：

- 前提 ： 使用随机置换哈希 $π$，保证键的低位可作为路由索引。

- 设定 $g*(x) = π(x) mod N$，也就是把键的前 $ log N$  位直接用作顶层路由。
- 在小隔间中，数组 S 不再存储这 $log N$ 位，只存储键的剩余高位和值。
- 为了在查询时恢复出完整的 $π(x)$，我们在每个槽的 Meta 中额外记录了修复信息：
    - $delta = g_I(x) - i$。由于 $g_I(x) $是键在 cubby 内的随机偏好槽位，i 是实际存储槽位，差值恢复了 $g_I(x)$。
    - mid_bits：x 中从$ log|I|+1 $到 $log K $的那部分比特，用来拼出完整的 $g*$ 的低位。
- 有了 $g_I(x) $和 mid_bits，加上已知的设施编号 r（提供高位），就能完整重建 $g*(x)$（即 $π(x) $的低$ log N$ 位），再拼上 $S$ 中存储的高位，就得到完整的 $π(x)$，最后用 $π⁻¹$ 得到真实键。

这样一来，存储数组 S 中每个键只占用$log|U| - log N$ 比特，而不是 $log|U| $比特。加上路由器和 Meta 中的额外开销（均摊为 $O(log^{(k)} n))$，总空间接近信息论下界。

#### 3.6 总结算法全景

整个哈希表的算法逻辑可以总结为三个层次的组合：

1.  微观层（Cubby）：k-kick 树决定每个元素的实际存储位置，用 FreeSlotTree 快速找空位，所有动态信息编码进 MiniArray Meta，空间开销受控于探针复杂度。
2.  路由层（Local Query Router + D）：Trie 编码的本地路由器负责在 O(1) 时间内把查询键路由到正确的 cubby 和槽位，其空间开销与元素数量及探针复杂度匹配。
3.  宏观管理（Facility + Resize）：通过多层 cubby 结构和 tail 机制应对动态变化，通过后台重建来维护分布不变量，使得所有操作的时间保证在 O(k)，空间保持在 $n log(|U|/n) + O(n log^(k) n) $比特。

> 与本文代码的关系：以上为论文级算法全景。仓库内 C++ 实现在 Facility、商压缩、双表迁移、分布重建上与上面对齐；k-kick 树本体以及 Router、MiniArray 的论文级 O(1) 微观结构为工程简化，详见下一节「系统设计」中的「实现落点与理论差异」。

## 系统设计

- 查询：目标 $O(1)$（Patricia 深度受哈希位数限制；每桶键数为 polylog 量级）
- 插入与删除：k-kick 链长度上界由 TableParams::k 给出；删除在非尾 cubby 上另含至多一次从 tail 搬入
- 空间：slots 存 quotient payload，MiniArray 存 MetaEntry，Router 维护编码比特长度供 metrics

```cpp
        return i;
    }
    return 0;
  }
```

MiniArray 的更新语义（include/otsh/mini_array.h）：

```cpp
// MiniArray（可工作的正确版）
// - 变长 bit entry，紧凑打包在一个 bit buffer 中
// - 更新时允许 O(m) 重打包（m 为该 MiniArray 的 entry 数）
class MiniArray {
public:
  using Bits = std::vector<uint64_t>; // little-endian 64-bit words 小端64位字

  MiniArray() = default;
  explicit MiniArray(size_t n) { reset(n); }

  void reset(size_t n) {
    bitlens_.assign(n, 0);
    offsets_.assign(n + 1, 0);
    entries_.assign(n, Bits{});
    buf_.clear();
  }

  size_t size() const { return bitlens_.size(); }

  // 返回总 bit 数（用于 metrics / 空间估计）。
  uint64_t bits_total() const { return offsets_.empty() ? 0 : offsets_.back(); }

  uint32_t bitlen(size_t i) const { return bitlens_.at(i); }

  // 读取 entry i 的 bits（按 64-bit words 返回；多余高位为 0）。
  Bits access(size_t i) const {
    const uint32_t bl = bitlens_.at(i);
    const uint64_t start = offsets_.at(i);
    Bits out((bl + 63) / 64, 0);
    for (uint32_t b = 0; b < bl; b++) {
      if (get_bit(start + b))
        out[b / 64] |= (1ULL << (b % 64));
    }
    return out;
  }

  void update(size_t i, const Bits &bits, uint32_t bitlen) {
    if (i >= bitlens_.size())
      throw std::out_of_range("MiniArray::update");
    bitlens_[i] = bitlen;
    entries_[i] = bits;
    repack();
  }
```

伪代码（MiniArray）：

    update(i, bits, len) → 写 bitlens[i]、entries[i]；repack() 全表重打包到 buf_

```cpp
      return r;

      const Facility &of = facility_for_key(*old_, gx);
      const size_t ob = route_bucket_for(*old_, gx);
      auto [oloc, osteps] = router_at(of, ob).locate(key);
      r.router_probe_steps = std::max(r.router_probe_steps, osteps);

                                facility_index_of(*old_, &of), *old_, gx);
      if (r.found && oloc.has_value())
        r.cubby_tier = oloc->first->tier;
    }
    return r;
  }
```

```cpp
  InsertResult insert_no_lock(uint64_t key, bool /*persist_semantics*/) {
    InsertResult r;
    if (active_.facilities.empty()) {
      r.ok = false;
      r.error = "not_initialized";
      return r;
    }
    uint64_t gx = pi_.pi(key);
    Facility &f = facility_for_key(active_, gx);
    const size_t b = route_bucket_for(active_, gx);
    if (auto loc = router_at(f, b).locate(key); loc.first.has_value()) {
      r.ok = true;
      r.inserted = false;
      r.router_probe_steps = loc.second;
      r.kick_count = 0;
      r.cubby_tier = -1;
      const uint64_t meta_bits = facility_router_bits_sum(f);
      global_metrics().on_insert(/*moved=*/0, /*router_steps=*/loc.second,
                                 /*meta_bits=*/meta_bits);
      return r;
    }
    if (old_) {
      Facility &of = facility_for_key(*old_, gx);
      const size_t ob = route_bucket_for(*old_, gx);
      if (auto oloc = router_at(of, ob).locate(key); oloc.first.has_value()) {
        r.ok = true;
        r.inserted = false;
        r.router_probe_steps = oloc.second;
        r.kick_count = 0;
        r.cubby_tier = -1;
        const uint64_t meta_bits = facility_router_bits_sum(f);
        global_metrics().on_insert(/*moved=*/0, /*router_steps=*/oloc.second,
                                   /*meta_bits=*/meta_bits);
        return r;
      }
    }
    ensure_tail(active_, f);

    // Step D: k-kick insertion into tail cubby
    auto kr = kkick_insert(f, f.tail, key,
                           facility_index_of(active_, &f));
    if (!kr.ok) {
      r.ok = false;
      r.error = "tail_full";
      r.router_probe_steps = 0;
      r.kick_count = 0;
      r.cubby_tier = f.tail ? f.tail->tier : -1;
      return r;
    }
    n_++;
    r.ok = true;
    r.inserted = true;
    r.kick_count = kr.moved;
    auto [__, steps] = router_at(f, b).locate(key);
    r.router_probe_steps = steps;
    r.cubby_tier = f.tail ? f.tail->tier : -1;
    const uint64_t meta_bits =
        facility_router_bits_sum(f) + f.tail->meta.bits_total();
    global_metrics().on_insert(/*moved=*/kr.moved, /*router_steps=*/steps,
                               /*meta_bits=*/meta_bits);
    maybe_schedule_rebuild(f);
    return r;
  }
```

```cpp
  bool slot_pi_matches(const Cubby &c, size_t slot, size_t facility_r,
                       const TableState &t, uint64_t key_gx) const {
    if (slot >= c.slots.size() || !c.slots[slot].has_value())
      return false;
    MetaEntry me;
    if (!decode_slot_meta(c, slot, t, me))
      return false;
    const auto gx = reconstruct_gx_pi(facility_r, t.N, t.K, c.capacity, slot,
                                      *c.slots[slot], me, logn_of(t));
    return gx.has_value() && *gx == key_gx;
  }

  std::optional<uint64_t> recover_key_from_slot(const Cubby &c, size_t slot,
                                                size_t facility_r,
                                                const TableState &t) const {
    if (slot >= c.slots.size() || !c.slots[slot].has_value())
      return std::nullopt;
    MetaEntry me;
    if (!decode_slot_meta(c, slot, t, me))
      return std::nullopt;
    const auto gx = reconstruct_gx_pi(facility_r, t.N, t.K, c.capacity, slot,
                                        *c.slots[slot], me, logn_of(t));
    if (!gx.has_value())
      return std::nullopt;
    return pi_.inverse(*gx);
  }

  void put_quotient_slot(Cubby &c, size_t slot, uint64_t key, uint64_t gx_pi,
                         size_t facility_r, const TableState &t, uint32_t ins) {
    (void)facility_r;
    const uint32_t ln = logn_of(t);
    const MetaCodecLayout ly = meta_layout(t.K, c.capacity);
    const MetaEntry me = make_meta_entry(key, gx_pi, slot, facility_r, t.N, t.K,
                                         c.capacity, ins);
    auto pr = encode_meta_entry(me, ly);
    c.slots[slot] = quotient_payload(gx_pi, ln);
    c.meta.update(slot, pr.first, pr.second);
  }
```

伪代码（槽校验 / 恢复 / 写入）：

    slot_pi_matches → decode meta，reconstruct gx，与查询 gx 比较
    recover_key_from_slot → reconstruct 后 π⁻¹ 得原 key
    put_quotient_slot → make+encode meta，slots[slot]←quotient_payload，meta.update

```cpp
  size_t probe_slot(uint64_t key, size_t cap, uint64_t i) const {
    // 伪随机探针序列：splitmix64(key_bits ^ i) % cap
    // 先用 pi(key) 打散，避免 key 低位模式影响。
    const uint64_t gx = pi_.pi(key);
    const uint64_t h = splitmix64(gx ^ (i * 0x9e3779b97f4a7c15ULL));
    return static_cast<size_t>(cap == 0 ? 0 : (h % cap));
  }
```

```cpp
  KickInsertResult kkick_insert(Facility &f, Cubby *c, uint64_t key,
                                size_t facility_r) {
    KickInsertResult out;
    if (!c || c->capacity == 0) {
      out.ok = false;
      return out;
    }

    const uint64_t kmax = static_cast<uint64_t>(std::max(0, params_.k));
    uint64_t cur_key = key;

    auto router_erase_key = [&](uint64_t ky) {
      router_at(f, route_bucket_for(active_, pi_.pi(ky))).erase(ky);
    };
    auto router_put_key = [&](uint64_t ky, Cubby *cb, size_t slot) {
      router_at(f, route_bucket_for(active_, pi_.pi(ky)))
          .insert(ky, std::make_pair(cb, slot));
    };

    for (uint64_t step = 0; step <= kmax; step++) {
      const size_t pos = probe_slot(cur_key, c->capacity, step);
      if (!c->slots[pos].has_value()) {
        c->occupied.push_back(pos);
        c->size++;
        const uint64_t gxc = pi_.pi(cur_key);
        put_quotient_slot(*c, pos, cur_key, gxc, facility_r, active_,
                          static_cast<uint32_t>(step));
        c->free_slots.mark_used(static_cast<int>(pos));
        router_put_key(cur_key, c, pos);
        out.ok = true;
        out.slot = pos;
        out.moved = step;
        return out;
      }

      const auto kicked_opt =
          recover_key_from_slot(*c, pos, facility_r, active_);
      if (!kicked_opt) {
        out.ok = false;
        return out;
      }
      const uint64_t kicked_key = *kicked_opt;

      router_erase_key(kicked_key);
      const uint64_t gxc = pi_.pi(cur_key);
      put_quotient_slot(*c, pos, cur_key, gxc, facility_r, active_,
                        static_cast<uint32_t>(step));
      router_put_key(cur_key, c, pos);
      cur_key = kicked_key;
    }

    auto free = cubby_find_free(*c);
    if (!free) {
      out.ok = false;
      return out;
    }
    const size_t pos = *free;
    c->occupied.push_back(pos);
    c->size++;
    const uint64_t gxc = pi_.pi(cur_key);
    put_quotient_slot(*c, pos, cur_key, gxc, facility_r, active_,
                      static_cast<uint32_t>(kmax + 1));
    c->free_slots.mark_used(static_cast<int>(pos));
    router_put_key(cur_key, c, pos);
    out.ok = true;
    out.slot = pos;
    out.moved = kmax + 1;
    return out;
  }
```

伪代码（kkick_insert）：

    for step = 0 .. kmax：
        pos ← probe_slot(cur, cap, step)
        若 slots[pos] 空 → put_quotient、router 登记 cur，return moved=step
        否则踢出 pos 原键，路由器改指向，cur ← 被踢键继续
    若仍剩 cur → cubby_find_free 找另一空位安置，moved=kmax+1

```cpp
  void ensure_tail(TableState &t, Facility &f) {
    if (f.tail && f.tail->size < f.tail->capacity)
      return;

    // 如果旧 tail 已满，把它作为“满 cubby”挂回 tiers[tail_tier]。
    if (f.tail_owned && f.tail_owned->size == f.tail_owned->capacity) {
      if (static_cast<int>(f.tiers.size()) <= f.tail_tier)
        f.tiers.resize(static_cast<size_t>(f.tail_tier + 1));
      f.tiers[static_cast<size_t>(f.tail_tier)].push_back(
          std::move(f.tail_owned));
      f.tail = nullptr;

    // 新建一个 tail（默认 tier=0）。
    f.tail_tier = 0;
    auto c = std::make_unique<Cubby>();
    c->tier = f.tail_tier;
    c->capacity = cubby_capacity(t.K, t.N, c->tier);
    c->size = 0;
    c->slots.assign(c->capacity, std::nullopt);
    c->occupied.clear();
    c->occupied.reserve(c->capacity);
    c->meta.reset(c->capacity);
    init_cubby_free_slots(*c);

    f.tail = c.get();
    f.tail_owned = std::move(c);
```

分布目标与 rebuild 调度（src/ht.cpp）：

```cpp
  uint64_t target_tier_count(int j) const {
    if (j < 0)
      return 0;
    const double n = std::max<double>(2.0, static_cast<double>(std::max<uint64_t>(1, n_)));
    const double a = iter_log2(n, j);
    const double b = iter_log2(n, j + 1);
    const double tj = (a * a) / std::max(1.0, b * b);
    return static_cast<uint64_t>(std::max(1.0, std::floor(tj)));
  }

  void maybe_schedule_rebuild(Facility &f) {
    for (int j = 0; j < f.max_tier; j++) {
      if (static_cast<int>(f.tiers.size()) <= j)
        continue;
      const uint64_t tj = target_tier_count(j);
      const uint64_t cnt =
          static_cast<uint64_t>(f.tiers[static_cast<size_t>(j)].size());
      if (cnt > 3 * tj) {
        Facility *pf = &f;
        scheduler_.enqueue([this, pf, j]() { this->rebuild_down(*pf, j); });
      } else if (tj >= 2 && cnt < tj / 2) {
        const int j2 = j + 1;
        if (static_cast<int>(f.tiers.size()) > j2 &&
            !f.tiers[static_cast<size_t>(j2)].empty()) {
          Facility *pf = &f;
          scheduler_.enqueue([this, pf, j]() { this->rebuild_up(*pf, j); });
        }
      }
    }
  }

  void run_rebuild_budget() { scheduler_.step_budget(1); }
```

伪代码（分布调度）：

    tj ← target_tier_count(j)（迭代 log 公式）
    cnt ← tiers[j].size()
    若 cnt > 3×tj → enqueue rebuild_down(f,j)
    若 tj≥2 且 cnt < tj/2 且 tiers[j+1] 非空 → enqueue rebuild_up(f,j)
    run_rebuild_budget() → 每次操作最多执行 1 个队列任务

rebuild_down 与 rebuild_up 主体（src/ht.cpp）：

```cpp
  void rebuild_down(Facility &f, int j) {
    if (j < 0 || j >= f.max_tier)
      return;
    if (static_cast<int>(f.tiers.size()) <= j)
      return;
    const uint64_t tj = target_tier_count(j);
    auto &tier = f.tiers[static_cast<size_t>(j)];
    if (tier.size() < tj || tj == 0)
      return;

    const int j2 = j + 1;
    if (static_cast<int>(f.tiers.size()) <= j2)
      f.tiers.resize(static_cast<size_t>(j2 + 1));

    const size_t fi = facility_index_of(active_, &f);
    std::vector<uint64_t> keys;
    for (uint64_t i = 0; i < tj; i++) {
      auto c = std::move(tier.back());
      tier.pop_back();
      for (size_t si = 0; si < c->slots.size(); ++si) {
        if (!c->slots[si].has_value())
          continue;
        if (auto rk = recover_key_from_slot(*c, si, fi, active_); rk)
          keys.push_back(*rk);
      }
    }

    auto newc = create_cubby(active_.K, active_.N, j2);
    for (uint64_t k : keys) {
      auto free = cubby_find_free(*newc);
      if (!free)
        break;
      const size_t pos = *free;
      newc->occupied.push_back(pos);
      newc->size++;
      newc->free_slots.mark_used(static_cast<int>(pos));
      const uint64_t gkx = pi_.pi(k);
      put_quotient_slot(*newc, pos, k, gkx, fi, active_, 0);
      const size_t bk = route_bucket_for(active_, gkx);
      router_at(f, bk).erase(k);
      router_at(f, bk).insert(k, std::make_pair(newc.get(), pos));
    }
    f.tiers[static_cast<size_t>(j2)].push_back(std::move(newc));
    global_metrics().on_rebuild_down();
  }

  // 设计文档：tier j+1 拆成多个 tier j（create rebuild）。
  void rebuild_up(Facility &f, int j) {
    const int j2 = j + 1;
    if (j < 0 || j2 > f.max_tier)
      return;
    if (static_cast<int>(f.tiers.size()) <= j2)
      return;
    auto &upper = f.tiers[static_cast<size_t>(j2)];
    if (upper.empty())
      return;

    const uint64_t tj = target_tier_count(j);
    if (tj < 2)
      return;

    auto big = std::move(upper.back());
    upper.pop_back();

    const size_t fi = facility_index_of(active_, &f);
    std::vector<uint64_t> keys;
    keys.reserve(big->size);
    for (size_t si = 0; si < big->slots.size(); ++si) {
      if (!big->slots[si].has_value())
        continue;
      if (auto rk = recover_key_from_slot(*big, si, fi, active_); rk)
        keys.push_back(*rk);
    }
    for (uint64_t k1 : keys) {
      const size_t bk = route_bucket_for(active_, pi_.pi(k1));
      router_at(f, bk).erase(k1);
    }
    big.reset();

    if (static_cast<int>(f.tiers.size()) <= j)
      f.tiers.resize(static_cast<size_t>(j + 1));

    const size_t nj = static_cast<size_t>(tj);
    std::vector<std::unique_ptr<Cubby>> news;
    news.reserve(nj);
    for (size_t i = 0; i < nj; ++i)
      news.push_back(create_cubby(active_.K, active_.N, j));

    for (size_t i = 0; i < keys.size(); ++i) {
      const uint64_t k1 = keys[i];
      Cubby *target = news[i % news.size()].get();
      auto fs = cubby_find_free(*target);
      if (fs) {
        const size_t pos = *fs;
        target->occupied.push_back(pos);
        target->size++;
        target->free_slots.mark_used(static_cast<int>(pos));
        const uint64_t gkx = pi_.pi(k1);
        put_quotient_slot(*target, pos, k1, gkx, fi, active_, 0);
        const size_t bk = route_bucket_for(active_, gkx);
        router_at(f, bk).insert(k1, std::make_pair(target, pos));
      } else {
        (void)kkick_insert(f, target, k1, fi);
      }
    }
    for (auto &c : news)
      f.tiers[static_cast<size_t>(j)].push_back(std::move(c));
    global_metrics().on_rebuild_up();
  }
```

伪代码（rebuild_down）：

    从 tier[j] 连续弹出 tj 个 cubby，收集全部 key
    新建一个 tier[j+1] cubby，逐键 put_quotient 并更新各 key 的 router

伪代码（rebuild_up）：

    从 tier[j+1] 弹出一个大 cubby，收集 key，清路由
    新建 tj 个 tier[j] cubby，轮转装键；单 cubby 满则 kkick_insert 兜底
    将新 cubby 们 push 回 tiers[j]

扩容与双表迁移（src/ht.cpp）：

```cpp
  void maybe_resize_locked() {
    if (active_.facilities.empty() || active_.N < 2)
      return;
    const double lf = (params_.load_factor > 0.0 && params_.load_factor < 1.0)
                          ? params_.load_factor
                          : 0.90;
    if (!old_ && static_cast<double>(n_) > static_cast<double>(active_.N) * lf) {
      start_migration(active_.N * 2);
    } else if (!old_ && n_ > 0 && n_ < active_.N / 4) {
      start_migration(std::max<uint64_t>(2, active_.N / 2));
    }
  }

  void start_migration(uint64_t newN) {
    // Phase7: 双表迁移（渐进）
    scheduler_.clear();
    global_metrics().on_resize_start();
    old_ = std::move(active_);
    active_.reset();
    migrate_progress_ = 0;

    active_.N = next_pow2(newN);
    active_.K = choose_K(active_.N);
    const uint64_t facilities_cnt = std::max<uint64_t>(1, active_.N / active_.K);
    active_.facilities.resize(static_cast<size_t>(facilities_cnt));
    for (auto &f : active_.facilities) {
      f.tiers.clear();
      f.max_tier = std::max(0, params_.k);
      f.D.assign(static_cast<size_t>(active_.K), Router{});
      f.tail = nullptr;
      f.tail_owned.reset();
      ensure_tail(active_, f);
    }
  }

  void run_migrate_budget() {
    if (!old_)
      return;
    if (migrate_progress_ >= old_->facilities.size()) {
      old_.reset();
      migrate_progress_ = 0;
      global_metrics().on_resize_finish();
      return;
    }
  }
```

伪代码（maybe_resize_locked）：

    若无旧表且 n > lf×N → start_migration(2N)
    若无旧表且 n < N/4 → start_migration(max(2, N/2))

伪代码（start_migration）：

    scheduler.clear；old ← move(active)；active 按 next_pow2(newN) 与 choose_K 重建
    每个新 facility：空 tiers、K 个空 Router、ensure_tail

伪代码（run_migrate_budget 开头）：

    若无 old_ → return
    若已迁完所有 facility → 清 old_、on_resize_finish；否则下文继续扫 keys 并重插入 active（见源 561 行起）

run_migrate_budget 后续在源文件中继续收集 keys 并重插入 active，全文见 src/ht.cpp 第 561 行起。

### Quotienting

quotient_payload、reconstruct_gx_pi、make_meta_entry 的实现摘录（src/meta_entry.cpp）：

```cpp
uint64_t quotient_payload(uint64_t gx_pi, uint32_t logN) {
  if (logN >= 64)
    return 0;
  return gx_pi >> logN;
}

std::optional<uint64_t> reconstruct_gx_pi(uint64_t facility_r, uint64_t N,
                                           uint64_t K, size_t cubby_capacity,
                                           size_t slot_index, uint64_t payload,
                                           const MetaEntry &m, uint32_t logN) {
  if (N < 2 || K == 0 || logN == 0 || logN >= 64)
    return std::nullopt;
  const size_t cap = std::max<size_t>(1, cubby_capacity);
  const int64_t g_i = static_cast<int64_t>(slot_index) + m.delta;
  if (g_i < 0 || static_cast<uint64_t>(g_i) >= cap)
    return std::nullopt;
  const uint64_t g_k =
      static_cast<uint64_t>(m.mid_bits) * cap + static_cast<uint64_t>(g_i);
  if (g_k >= K)
    return std::nullopt;
  const uint64_t g_star = facility_r * K + g_k;
  if (g_star != (g_star & (N - 1)))
    return std::nullopt;
  const uint64_t gx = (payload << logN) | g_star;
  return gx;
}

MetaEntry make_meta_entry(uint64_t key, uint64_t gx_pi, size_t slot_index,
                          uint64_t facility_r, uint64_t N, uint64_t K,
                          size_t cubby_capacity, uint32_t ins_tag) {
  MetaEntry e;
  (void)facility_r;
  (void)N;
  const size_t cap = std::max<size_t>(1, cubby_capacity);
  const uint64_t g_star = gx_pi & (N - 1);
  const uint64_t g_k = g_star & (K - 1);
  const uint64_t g_i = g_k % cap;
  e.delta = static_cast<int64_t>(g_i) - static_cast<int64_t>(slot_index);
  e.mid_bits = static_cast<uint32_t>(g_k / cap);
  const uint64_t h = splitmix64(key ^ gx_pi ^ 0x9e3779b97f4a7c15ULL);
  e.router_bits = static_cast<uint32_t>(h ^ (h >> 32)) & 0xffffu;
  e.insert_bits = ins_tag & 0xffu;
  return e;
}
```

伪代码（Quotienting 三函数）：

    quotient_payload(gx) → gx >> logN（存槽的高位商）
    reconstruct → g_i←slot+delta；g_k←mid_bits×|I|+g_i；g_star←r×K+g_k；gx←payload<<logN | g_star
    make_meta_entry → g_star←gx∧(N-1)；拆 g_k、g_i；写 delta、mid_bits、router_bits、insert_bits

### 实现落点与理论差异（审查结论）

| 论文组件 | 当前实现 | 说明 |
|----------|----------|------|
| k-kick 树（多层 cubby、s(x)、跨层驱逐） | 上列 kkick_insert 仅在单个 cubby 内有界踢键；include/otsh/kkick_tree.h 为占位 | 与论文 k-kick 树不等价，属高影响差异 |
| LocalQueryRouter（半字查找表、增量 O(1)） | Router：insert/erase 内全量 rebuild（见 src/router.cpp） | 正确性满足，更新代价高于论文目标 |
| MiniArray（字级 B-树 + 查找表） | update 调用 repack（见上列 mini_array.h） | 单槽更新与 cubby 容量线性相关 |
| 双表迁移与分布重建预算 | old_、RebuildScheduler、run_rebuild_budget 与 insert/erase 尾部调用 | 与论文叙述一致 |

解读 otsh_op_metrics 中的 probe_count、kick_count 时应考虑首行与次行差异。

### 复杂度总结

| 操作 | 论文目标 | 本实现备注 |
|------|----------|------------|
| query | $O(1)$ | Router::locate 沿 Patricia 走步，桶内线性扫描碰撞 |
| insert | $O(k)$ | kkick_insert 中 step 上界为 params_.k |
| delete | $O(k)$ | 非尾删除时另有一次 tail 搬移与两次路由器更新 |
| rebuild | 均摊 $O(1)$ | run_rebuild_budget 每操作 step_budget(1)；单次 rebuild 仍与 cubby 规模线性相关 |
